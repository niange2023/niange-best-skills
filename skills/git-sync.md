# 同步

## 目的
在多设备间无缝同步项目，自动执行 Git 拉取、提交、推送流程。

## 触发词

- "sync" / "同步" / "git同步"
- "推送" / "提交" / "提交并推送"
- "更新代码" / "拉取远端"

## 执行流程

### Step 1: 状态检查
```bash
git status
```

### Step 2: 拉取远端最新代码
```bash
git pull origin main
```
如果冲突，提示用户手动解决。

### Step 3: 提交本地修改
如果有未提交的修改：
```bash
git add .
git commit -m "chore: 本地积累修改 - $(date +'%Y%m%d %H:%M')"
```

### Step 4: 推送到远端
```bash
git push origin main
```

## 冲突处理

如果 `git pull` 时出现冲突：

1. **停止执行**，不继续提交和推送
2. **告知用户**：检测到冲突，需要手动解决
3. **提供解决步骤**：
   ```bash
   # 1. 查看冲突文件
   git status

   # 2. 手动编辑冲突文件（搜索 <<<<<<<）

   # 3. 标记解决
   git add 冲突文件
   git commit
   git push
   ```

## 完成反馈

同步完成后，简要报告：
- 是否有拉取到远端更新
- 提交了哪些文件
- 是否推送成功

## 示例对话

```
用户: 同步
智子: 正在与 GitHub 同步...
      [拉取] 远端已是最新
      [提交] 3 个文件已修改
      [推送] 成功
      ✅ 同步完成
```
